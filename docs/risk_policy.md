# Risk and Compliance Considerations

This concept is designed as an information service, not an investment recommendation service.

## Principles
1. Do not guarantee returns.
2. Do not present alerts as buy/sell recommendations.
3. Clearly state that crypto assets are volatile and may result in loss.
4. Use public market and on-chain data only.
5. Do not store private keys, API secrets, or user credentials.
6. Separate service alerts from automated trading execution.
7. Evaluate user fatigue through alert opt-out and click-through data.
